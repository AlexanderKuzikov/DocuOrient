# DocuOrient

Определение правильной ориентации текста на выровненном документе.  
Часть проекта **DocuMind** — интеллектуальной обработки юридических документов.

## Что делает

Принимает на вход уже выпрямленный grayscale-документ (результат `DocuDeskew`) и определяет, нужно ли довернуть его на 90°, 180° или 270°, чтобы текст читался правильно. Затем выполняет поворот. Основан на анализе проекций текста, без использования OCR и нейросетей.

## Возможности

- Вход: выровненное grayscale-изображение (PNG).
- Определяет ось строк (горизонтальную/вертикальную) по текстурному профилю.
- Различает верх/низ (0° или 180°) и лево/право (90° или 270°) через асимметрию яркости.
- Устойчив к документам с малым количеством текста — при низкой уверенности оставляет без изменений.
- Быстрый и детерминированный.

## Алгоритм (кратко)

1. Бинаризация (Otsu).
2. Вычисление горизонтальной проекции для двух положений (0° и 90°), оценка текстурности (СКО).
3. Выбор оси строк (горизонтальная/вертикальная).
4. Определение «верха» через сравнение яркости верхней/нижней трети (или левой/правой).
5. Доворот через `sharp`.

## Установка

```bash
npm install docu-orient
```

## API

```js
const { orient } = require('docu-orient');

const result = await orient(deskewedImageBuffer, options);
// {
//   rotationStep: 0 | 90 | 180 | 270,
//   orientedImage: Buffer   // финальное grayscale PNG
// }
```

### Параметры options (необязательные)

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `otsuThreshold` | `boolean` | `true` | Использовать ли Otsu для бинаризации |
| `minTextureScore` | `number` | `5.0` | Минимальный порог текстурности (ниже — без поворота) |

## Пример использования

```js
const fs = require('fs');
const { orient } = require('docu-orient');

(async () => {
  const input = fs.readFileSync('straight.png');
  const { orientedImage, rotationStep } = await orient(input);
  console.log(`Применён доворот: ${rotationStep}°`);
  fs.writeFileSync('final.png', orientedImage);
})();
```
